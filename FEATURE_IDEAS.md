# SollyCricket — Prospective Feature Ideas

> A living document of ideas, grouped by theme. Add entries as they come up; pick from the list when scoping the next round of work.

---

## 🖥️ UI & User Experience

- **Live match status bar** — A slim bar at the top showing current run rate, required run rate (2nd innings), and session summary (overs bowled, wickets lost).
- **Field map (live, every ball)** — A cricket-field SVG showing all 11 fielding positions laid out on a 2D overhead view. On each delivery, an animated ball-trajectory line draws from the pitch centre to where the ball went — straight to a fielder if caught , through the infield for a single/double, or all the way to the boundary rope for a 4/6. The trajectory line colour encodes the outcome, with a legend beside the map. The map resets / fades the previous ball so you can always see the current delivery clearly. Updates automatically every ball.

- **Wagon wheel (accumulated)** — A separate view (on summary tab) showing the accumulated shot distribution for a selected scope:
  - **Current over** — all balls of the current over overlaid as faint markers, plus the last ball highlighted.
  - **This batter** — all scoring shots by the current striker, showing their scoring areas.
  - **This innings** — full team wagon wheel.
  - Uses the same colour legend as the live field map so the mapping is consistent.
- **Manhattan** - A separate view (on summary tab) showing a manhattan graph of the number of runs and wickets each over.
- **Partnership tracker** — Highlight the current partnership runs and balls faced, shown between the scorecard and the commentary feed.
- **Dark mode toggle** — Respect `prefers-color-scheme` or add a manual toggle in the header.
- **Responsive / resizable layout** — Allow dragging the divider between scorecard and commentary, or auto-collapse to stacked view on narrow windows.

## 🏏 Commentary & Narrative

- **End-of-over summary** — After the 6th ball of an over, insert colour commentary: runs scored off the over, wickets fallen, bowler's figures so far, and how it shifts the match balance.
- **End-of-innings summary** — Brief paragraph summarising the innings: top scorer, strike rate, key partnerships, bowling highlights.
- **End-of-match presentation** — After the match ends, show:
  - A one-paragraph match summary
  - **Player of the Match** selection (based on highest impact — most runs, most wickets, or all-round performance)
  - "Interview answers" from the winning captain, losing captain, and the Player of the Match (generated by LLM or picked from template quips)
- **Session / drinks-break commentary** — After every ~10 overs (or at a natural pause), insert a session summary: "Session 1 belonged to Australia — 145/3 at a run rate of 4.2."
- **Player backstories / flavour** — Brief one-liner about a player when they come to the crease or take a wicket (e.g. "Starc, who's had a lean series so far…"). Store a `blurb` field in the player model.
- **Umpire decisions / DRS** — Add a small delay with "Umpire gives it out… batter reviews… review upheld/reversed" narrative beats.

## ⚙️ Gameplay & Simulation

- **Player roles for user-set teams** — When users enter custom team/player names on the setup page, keep the fixed role slots (3 openers, 4 middle-order, 1 all-rounder, 1 wk, 1 fast bowler, 1 spinner) so the simulation still works sensibly.
- **Injuries / substitution** — Rare chance a player gets injured mid-match; substitution from a limited bench.
- **Pitch / weather conditions** — A pitch type (flat, green, dusty, cracked) and weather (sunny, overcast, drizzle) that bias batting/bowling/draw outcomes. Would need a new `MatchCondition` model and modifications to `simulate_delivery`.
- **Player form / confidence** — A form slider (hot / cold) that decays or improves over the match, affecting the `bat_rating` and `bowl_rating` used during simulation.
- **Manual over-rate / aggression settings** — User can nudge "aggression" (risk-taking) which shifts the probability distribution of shot outcomes.
- **Super Over / tie-breaker** — When scores are level after both innings, a 1-over-per-side eliminator.

## 🔌 Technical & Architecture

- **Offline-first commentary** — Build a richer local template engine (mix-and-match sentence parts, player stat injection) so the LLM gap feels smaller when the backend is cold-starting or offline.
- **Local LLM support (via Ollama)** — Already partially listed in the commentary providers; wire it into the setup page as a toggle so users running Ollama locally don't need the Render backend at all.
- **Match replay / timeline scrubber** — Store each ball's full state snapshot, then let the user scrub through the match timeline after it finishes.
- **Save / load match state** — Serialise `MatchState` to JSON so a match can be paused and resumed across app restarts.
- **Export match scorecard** — Generate a shareable image or HTML summary after the match ends (Cricinfo-style scorecard).
- **Configurable commentary provider** — A dropdown in the UI (or a config file) to pick which of the 5+ LLM providers to use, instead of auto-detecting via env vars.

## 🧪 Testing & Quality of Life

- **Deterministic seed replay** — Accept a random seed so a specific match sequence can be replayed identically (useful for debugging and for users who want to re-live a crazy finish).
- **Automated match simulation (no-GUI mode)** — Run N simulated matches head-to-head and spit out stats (average scores, win rates, most runs/wickets). Good for balance tuning.
- **Ball-by-ball logging to file** — Write every delivery's details to a log file so non-reproducible bugs can be diagnosed after the fact.
