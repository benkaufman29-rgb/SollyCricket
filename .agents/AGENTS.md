# Workspace Rules: Cricket Simulation

## Guidelines
- **Project Goals:** Always reference [PROJECT_CONTEXT.md](file:///C:/Users/Ben/coding/project-solly-cricket/PROJECT_CONTEXT.md) for the core architecture, gameplay logic, UI/UX aesthetics, and implementation roadmap.
- **Tech Stack:** Use Python 3.x, `pywebview` for the window shell, and local HTML5/Vanilla CSS3/ES6+ JavaScript for the frontend.
- **UI Design System:** Match ESPNcricinfo's clean blue/white palette, detailed scorecard layout, and scrollable commentary feed with distinct ball run bubbles. Make it look professional and highly stylized with responsive grids and panels.
- **Modularity:** Keep simulation logic (Match/Innings states) decoupled from UI rendering logic to allow eventually swapping the template commentary system with an LLM generator easily.
